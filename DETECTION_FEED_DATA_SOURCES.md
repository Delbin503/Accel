# Detection Feed — Data Sources Reference

Maps every visible field in the Detection Feed module to its real-world source module.

---

## Severity Levels

Severity is assigned by the **Rules Library** at the time a rule triggers. Each rule has a severity level set by the operator when the rule is created.

| Level | When it shows | Example triggers |
|---|---|---|
| **Critical** | Rule is set to `critical` — immediate threat, zero tolerance violations | Unauthorized asset movement with no access record, restricted zone entry with no credential match, weapon direction violation |
| **Medium** | Rule is set to `medium` — requires review but not immediate action | Loitering beyond dwell time threshold, unidentified person near restricted area, unauthorized presence without confirmed breach |
| **Low** | Rule is set to `low` — informational, logged for audit | Unattended object below the critical dwell threshold, minor SOP deviation, object stationary within logging window only |

Source: **Rules Library** — severity field on each rule (`Rul_001`, `Rul_002`, etc.)

|---|---|---|---|

| Detection type label | Category label for the triggered violation | `Asset movement — unauthorized` | **Model Management** (use-case definition) |
| Summary text | AI-generated short description of the event | `Asset AST-WPN-0421 exited Armoury-B at 14:23...` | **Detection Engine** / VLM output |
| `[[AST-WPN-0421]]` ref chip | Asset ID embedded in summary text | `AST-WPN-0421` | **CV Side** |
| `[[PER-U-7782]]` ref chip | Person/subject ID embedded in summary text | `PER-U-7782` | **CV Side** |
| Site name | Physical site where the camera is located | `Sembawang Naval` | **Site Overview** |
| Area name | Zone within the site | `Armoury-B` | **Site Overview → Zones** |
| Camera ID | Camera that captured the event | `Cam-12` | **Cameras** |
| Timestamp | Time the event was detected | `14:23:08` | **Detection Engine** (VMS) |
| Date | Calendar date of the event | `19 May 2026` | **Detection Engine** (VMS) |
| Thumbnail (CCTV frame) | Video frame at time of detection | gradient placeholder | **Live Monitoring** (CCTV stream) |
| Bounding box overlays | Detected object positions and labels in frame | `Person · 0.920`, `Asset · 0.960` | **Detection Engine** (VMS) |
| `Escalated` badge | Status indicator when event has been linked to a case | `Escalated` | **Incident Cases** (status written back) |
| `View Case →` button | Navigates to the linked incident case | `CASE-2026-0142` | **Incident Cases** |

## Event Drawer (click any event card)

### Header
| Field | What it is | Source |
|---|---|---|
| Severity badge | Same as card | **Rules Library** |
| Detection type title | Same as card | **Model Management** |
| Event ID · Model ID · Date · Time | Identifiers | **Detection Engine** / **Model Management** |

### Event Summary
| Field | What it is | Source |
|---|---|---|
| Summary text | Full AI-generated event description | **Detection Engine** / VLM output |
| `[[Entity]]` chips | Clickable asset / person / vehicle references | **CV Side** |

### VLM Reasoning
| Field | What it is | Source |
|---|---|---|
| Narrative reasoning text | Scene-level AI description from the Vision Language Model | **Detection Engine** (VLM — runs on the 8s clip) |

### Detected By (Model Card)
| Field | What it is | Source |
|---|---|---|
| Model name | Name of the model that triggered the detection | **Model Management** |
| Model ID | Use-case model identifier (e.g., `Mdl_002`) | **Model Management** |
| Trained date | Date the model version was last trained | **Model Management** |
| Last eval mAP | Mean Average Precision from latest model evaluation | **Model Management** |

### Detection Details Grid
| Field | What it is | Source |
|---|---|---|
| Model ID | Same as above | **Model Management** |
| Detection Type | Use-case category label | **Model Management** |
| Asset ID | Tracked asset identifier | **CV Side** |
| Person ID | Tracked subject identifier | **CV Side** |
| Area | Physical zone where event occurred | **Site Overview → Zones** |
| Camera | Camera that captured the event | **Cameras** |
| Site | Physical site | **Site Overview** |

### Detection Confidence Section
| Field | What it is | Source |
|---|---|---|
| Entity score cards (e.g., `Person Box · 0.920`) | Per-entity detection confidence score, parsed from bounding box label | **Detection Engine** (VMS) |
| Entity ID sub-label (e.g., `PER-U-7782`) | Cross-reference ID shown under confidence | **CV Side** |
| **Model Evaluation Performance** (collapsed) | | |
| Precision / Recall / F1 / mAP@0.5 | Aggregate model evaluation metrics | **Model Management** (last eval run) |
| Test frames count | Number of frames used in evaluation | **Model Management** |
| Eval date | When evaluation was last run | **Model Management** |
| Conf threshold / IoU threshold | Detection trigger thresholds | **Model Management** |
| Data split bar (Train / Val / Test %) | Dataset split percentages used in training | **Model Management** |

### Entities Involved Section
| Field | What it is | Source |
|---|---|---|
| Entity type chip (Person / Asset / Vehicle) | Kind of tracked entity | **CV Side** |
| Entity ID | Unique identifier | **CV Side** |
| Person name or description | Name if identified, or physical description | **CV Side** |
| Re-ID confidence | How confident the system is in the identity match | **CV Side** (Re-ID engine) |
| Total detections count | How many times this entity has been detected | **CV Side** |
| `View Info →` link | Opens full entity profile drawer | **CV Side** |

## Escalate Modal (Escalate Case button)

| Field | What it is | Source |
|---|---|---|
| Case title (pre-filled) | Auto-generated from event type + site + camera | Derived from the event |
| **Severity** picker | User selects case severity | UI interaction — defaults to event severity |
| **Assign To** search | Dropdown of team members to assign the case to | **User Management** |
| **SLA Target** (Acknowledge / Initial Action / Resolution) | Response time targets based on selected severity | Hardcoded SLA policy — not from any module |
| Initial Notes field | Free-text notes for the investigator | User input |

## Notes on Mismatches (Current Mock vs. Real System)

| Issue | Detail |
|---|---|
| **Model IDs are inconsistent** | Model Management shows `Mdl_001 = SOP Compliance`, `Mdl_002 = Muzzle Detection Protocol`. Detection Feed shows `Mdl_002 = Asset Movement Detection`, `Mdl_005 = Restricted Zone Monitor`, etc. These need to be reconciled when connecting to the real Model Management data. |
| **Model evaluation metrics are hardcoded** | The Precision/Recall/F1/mAP/data-split block in the Event Drawer is a static lookup table in the frontend code — not pulled from Model Management. This should become an API call to Model Management's last eval record. |
| **SLA targets are hardcoded** | The Acknowledge / Initial Action / Resolution times in the Escalate Modal are hardcoded constants, not configurable. These should be driven by a policy setting. |
| **Dismissed filter options are hardcoded subsets** | The Area and Model dropdowns in the Dismissed Events filter panel list only the areas/models that appear in the current mock data, not the full list from Site Overview or Model Management. |
| **Assignee list in Escalate Modal** | Currently sourced from the Incident Cases mock. In the real system this should be all users with access to the event's site, from User Management. |
